Безпека

Кожен запит захищений

JWT Bearer токени з терміном дії, bcrypt хешування з cost factor 12, PostgreSQL constraints на рівні БД, HTTPS на всіх ендпоінтах.

  • JWT Bearer авторизація на кожному запиті
  • bcrypt (cost=12) для хешування паролів
  • PostgreSQL CHECK та FOREIGN KEY constraints
  • HTTPS скрізь — Render забезпечує TLS
  • Ролі: client та admin з різними правами
1
POST /api/auth/login email + password
2
bcrypt.verify() password_hash порівняння
3
jwt.encode() user_id + exp → token
4
Authorization: Bearer <token> на кожен наступний запит

Мультиплатформенність

Один API — три клієнти

Єдиний REST API обслуговує PWA, Android-додаток та будь-який HTTP-клієнт.

PWA

Service Worker, offline-режим, installable на домашній екран. Vanilla JS, без фреймворків.

Android

Kotlin + Retrofit2 + OkHttp3. Bearer Auth Interceptor, Coroutines для асинхронних запитів.

REST Client

curl, Postman, будь-який HTTP-клієнт. Повна документація всіх 23 ендпоінтів.

Free tier Render — перший запуск ~30 сек

Terminal 
# 1. Логін
curl -X POST army-bank.onrender.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@army-bank.ua","password":"demo123"}'

# → {"token": "eyJhbGci..."}

# 2. Баланс
curl army-bank.onrender.com/api/user/balance \
  -H "Authorization: Bearer TOKEN"

# 3. Переказ
curl -X POST army-bank.onrender.com/api/transactions/transfer \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"to_email":"oleg@test.ua","amount":500}'

Часті питання

Додай Retrofit2 та OkHttp3 у build.gradle, створи ApiService інтерфейс з анотаціями @GET/@POST, налаштуй AuthInterceptor для Bearer токена. Базовий URL: https://army-bank.onrender.com/api/.

PWA отримує VAPID ключ через GET /api/push/vapid-public-key, підписується через PushManager.subscribe(), відправляє PushSubscription на POST /api/push/subscribe. Після транзакції — автоматичний push через pywebpush.

Кожна балансова операція — PostgreSQL-транзакція з SERIALIZABLE ізоляцією. Race condition виключений. Перед переказом: перевірка балансу, існування одержувача, валідності токена.
