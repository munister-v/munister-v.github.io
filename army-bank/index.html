05 // Безпека

Захист даних та контроль доступу

Сесійний токен, перевірка ролей, логування дій та валідація введення. Це демо, але підхід — як у справжньому продукті.

Сесії

Токен у заголовку `Authorization: Bearer …` з TTL, авто‑логін у браузері.

Ролі

Окремі інтерфейси: користувач / оператор / адміністратор з обмеженням доступу.

Аудит

Події: логін, реєстрація, операції, нарахування — у журналі аудиту.

FAQ

Питання та відповіді

Чому “вхід” не всередині лендінгу?

Лендінг розміщений на GitHub Pages (статичний хостинг). API та база працюють у застосунку банку. Тому кнопка “Відкрити банк” веде на /bank/.

Де адмін‑панель?

Усередині застосунку: відкрийте /bank/ → далі /admin (доступ за роллю admin).

Чи можна без редіректа, прямо під доменом?

Так, але потрібен проксі (наприклад Nginx на VPS або rules на хостингу). GitHub Pages сам по собі проксувати не вміє.
