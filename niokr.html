Группа разнородных устройств — серверы, GPU/NPU-блоки, edge-узлы — сама решает, на каком узле выгоднее выполнить запрос к нейросети. Без интернета. Без главного сервера.
Каждый узел равноправен и хранит локальную картину кластера. Сеть разорвалась — система продолжает работать.
  • A
    Автономность

    Каждый узел принимает решения локально без внешнего управляющего сигнала — даже при обрыве связи.

  • B
    Скорость

    Маршрутизация за микросекунды. Сложная оптимизация идёт в фоне и не блокирует обработку.

  • C
    Живучесть

    При жёстком отказе узла нагрузка переходит на тёплый резерв. Восстановление за ~2 с.

  • D
    Экономичность

    Весь демон — ~2 МБ RAM. Служебный трафик — 4,8 КБ/с. Работает в самом скромном контуре.

Закрытые промышленные, транспортные и оборонные контуры лишены постоянного интернета. Round-robin не знает ни о тепле GPU, ни о кэше модели, ни о глубине очереди.

  • 01
    Ограниченный канал

    Бюджет 64 кбит/с. Тяжёлый JSON-обмен исключён. Нужна компактная бинарная телеметрия с delta-кодированием.

  • 02
    Гетерогенное «железо»

    На кластере — разные CPU, GPU, NPU. Один планировщик обязан учитывать их все: память, тепло, очередь, кэш.

  • 03
    Split brain

    При разрыве каждый узел должен работать локально и восстанавливать консистентность без ручного вмешательства.

  • 04
    Air-gapped поставка

    Доставка образов и зависимостей без интернета требует воспроизводимых офлайн-сборок и строгого SBOM.

Модуль / функция
01
Node Agent
Демон узла. Читает конфигурацию, собирает метрики, участвует в gossip, регистрирует runtime-адаптеры.
02
Telemetry
32-байтный снимок: CPU/GPU/NPU, память, очередь, задержки, тепло, потери. Phi-accrual детектор.
03
State Bus
LWW-CRDT-карта кластера + опц. Raft для лиз-согласования. Работает при разрывах сети.
04
Scheduler L1/L2
L1 — скоринг за 0,33 мкс. L2 — MILP в фоне, подсказки по residency/standby/routing split.
05
Runtime Adapters
Единый trait: mock, ONNX Runtime, будущие GPU/NPU-бэкенды — без изменений ядра.
06
Zero-copy IPC
Тензоры через mmap shared memory — без копирования. 10 МБ за 0,58 мс через TensorHandle.
07
Recovery
Phi-accrual → standby-лизинг → повтор очереди → reroute. ~2,1 с без потери задач.
08
Dashboard
Локальная панель наблюдения кластера. Без CDN, офлайн, встроена в демон.

Классический IPC: сериализация → копирование → десериализация = десятки мс на 10+ МБ
Zero-copy: только хэндл (~100 байт), данные не двигаются
  • 1
    Выделение shm-региона

    Node-agent создаёт POSIX shared memory под тензор. Размер = shape × dtype. Имя — UUID.

  • 2
    Запись без копии

    Inference-движок пишет результат напрямую в mmap-область. Промежуточный буфер отсутствует.

  • 3
    Передача TensorHandle

    Через IPC уходит только лёгкий хэндл: имя + shape + dtype + offset. ~100 байт.

  • 4
    Чтение на месте

    Получатель маппирует тот же регион и работает напрямую. 10 МБ — 0,58 мс p99.

2,1

p95

Без ручного вмешательства. Задачи из очереди не теряются — идемпотентный повтор.
Phi-accrual φ Warm standby Raft-лизинг Идемпотентная очередь
  • φ
    Phi-accrual детектор

    φ вычисляется из исторических heartbeat-интервалов. При φ > threshold — узел «подозрительный». Нет ложных тревог при временных задержках сети.

  • S
    Захват standby-лизинга

    При confirmed failure резервный узел захватывает лизинг через Raft-консенсус. Гарантирует единственность standby.

  • R
    Повтор очереди

    Задачи «в полёте» повторяются на новом узле. Идемпотентность гарантирует корректность при возможном дублировании.

  • →
    Reroute трафика

    Планировщик исключает отказавший узел. Новые запросы уходят на живые узлы автоматически.

01

Сбор и кодирование телеметрии

Telemetry Layer: 32-байтный снимок. Delta-кодирование — только изменения сверх кванта, периодически полный снимок.

02

Gossip-обмен состоянием

State Bus распространяет дельты. CRDT — бесконфликтное слияние без лидера. Каждый узел видит актуальную картину.

03

Решение планировщика L1

Фильтрация подозрительных и перегруженных. Взвешенный скоринг. Выбор минимума стоимости. 0,33 мкс p99.

04

Исполнение через Runtime Adapter

Runtime Adapter запускает inference. Тензоры через Zero-copy IPC напрямую через shared memory. 0,58 мс p99 на 10 МБ.

05

Recovery при сбое

Phi-accrual фиксирует отказ → захват standby-лизинга → повтор идемпотентной очереди → reroute. ~2,1 с p95.

При разрыве сети — CRDT-копия работает локально, консистентность восстанавливается при возобновлении связи.

Ответы на вопросы · часто задаваемые
Главные вопросы
собственника
Вопрос
Ответ
Можно добавить новое железо?
Да. Любой узел с GPU/CPU/NPU — через trait-интерфейс без перекомпиляции ядра.
Что если узел упадёт?
Warm standby: p95 восстановления 2,1 с. Phi-accrual детектор без ложных срабатываний.
Нужен ли интернет после установки?
Нет. Система работает полностью в изолированной сети на скорости от 64 кбит/с.
Совместимость с нашим ПО?
Определяется на Этапе 1 (аудит). HTTP API + gRPC — интеграция с любым стеком.
Кто владеет кодом?
Полная передача прав заказчику. Исходники + лицензионный реестр в Этапе 5.
Это готовый продукт или НИОКР?
НИОКР с рабочим прототипом. Промышленная доводка — в Этапах 2–4. Все метрики уже достигнуты в симуляции.
Можно запустить на одном узле?
Да. Masterless = минимум 1 узел, максимум — не ограничен. Нет центрального сервера.

Схема · восстановление после отказа узла
Failover
за 2,1 с
Phi-accrual detector непрерывно отслеживает heartbeat-интервалы. Когда φ превышает порог — узел объявляется недоступным автоматически, без фиксированного таймаута.
Phi-accrual
Порог φ = 8 (настраивается). Нет false-positives при временных задержках сети.
Warm Standby
Резервный узел держит веса модели готовыми. Promotion без cold start — поэтому 2,1 с а не минуты.
